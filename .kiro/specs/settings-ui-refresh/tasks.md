# Implementation Plan

- [x] 1. バリデーションと一覧操作の純関数を抽出する
  - 既存設定画面のカスタムライブラリパス検証 (空 / 外部 URL / 絶対パス拒否) と、trim + 重複排除つき追加のロジックを純関数として切り出す
  - ユニットテストで既存実装と同一の判定結果 (空文字 / https: / app: / 絶対パス / 正常 vault 相対パスなど) が固定されている
  - _Requirements: 2.3, 2.4_
  - _Boundary: settings-ui_

- [x] 2. 設定タブの Setting API 再構築
- [x] 2.1 一覧以外の設定行とタブ骨格を Setting API で再実装する
  - React 実装を撤去し、一覧 (ライブラリ) 以外の全項目 (説明文・保存形式・真偽値 5 項目・drawio 表示言語・存在する場合は既定表示モード・外部同期の見出しと説明) を標準の設定行として構築する。ライブラリ一覧セクションは見出し枠のみ置き、行の描画・編集は 2.2 の専有とする
  - トップレベル見出しを出力せず、セクションは標準の設定見出しスタイルで表示し、全文言は既存 i18n を経由する (不足キーがあれば en/ja に追加する)
  - 設定画面向け CSS クラス (エラー文言用 .drawio-settings-error = var(--text-error) 等) を styles.css に追加する (インラインスタイル・innerHTML は使用しない)
  - 値変更が即座に永続化され、設定タブを閉じて再度開くと保存値が表示される
  - 一覧以外の全項目が標準の設定行構造で表示されている
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.1, 2.2, 2.5, 3.1, 3.2_
  - _Boundary: DrawioSettingTab, styles.css_
- [x] 2.2 ライブラリ一覧の編集 UI を実装する
  - 常設・カスタムライブラリ一覧をエントリ行 (削除ボタン付き) + 追加入力行 (Enter 対応) + 既定値リセット (常設のみ) で構成し、変更時は保存して再構築する (一覧の描画・編集は本タスクが全面所有)
  - 追加操作後に入力欄が空になり再フォーカスされる。重複追加は無視される
  - 不正パス入力時は 2.1 で追加済みの .drawio-settings-error クラスでエラー文言が追加行直下に表示され、一覧が変化しない
  - _Requirements: 2.1, 2.3, 2.4_
  - _Depends: 1_
  - _Boundary: DrawioSettingTab_
- [x] 2.3 テーマ追従の確認と残調整
  - ライト / ダークテーマ双方で設定画面の配色・レイアウトが破綻しないことを確認し、必要な CSS 調整を行う
  - インラインスタイル・innerHTML が設定タブ実装に存在しないことを確認する
  - _Requirements: 1.4_
  - _Boundary: DrawioSettingTab, styles.css_

- [x] 3. 設定画面の E2E テスト
  - 設定タブを開くと全項目が標準設定行として表示され、トップレベル見出しが存在しないことを検証する
  - toggle 変更 → 閉じて再度開くと値が保持されていることを検証する
  - ライブラリの追加 (Enter 含む)・削除・リセット・重複無視・不正パスのエラー表示を検証する
  - _Requirements: 1.1, 1.2, 1.4, 2.1, 2.2, 2.3, 2.4, 2.5_
